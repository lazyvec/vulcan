# Parcae PRD Draft (Phase 2)

## 0) Intake
- 아이디어 1줄: 복붙 템플릿이 아닌, 초정밀 연산 + LLM 기반의 '행동 결정(Action) 특화' 사주 엔진 (B2B/B2C 동시 트랙)
- 내부/외부: 외부 서비스 (B2B API 연동 + B2C 구독/단건 결제)
- 핵심 Edge: '불안 해소'와 '재물/궁합'에 맞춰진 고전환율(High-conversion) 0.04ms 명리 엔진. "언제 퇴사할까", "누구와 동업할까" 등 구체적 행동 타이밍을 지시.
- 제약(예산/일정/기술): 데드라인 "빠를수록 좋음", 제약/상한선 없음. 유지보수 및 서버비 Zero (Vercel Edge, Supabase 등 활용).

## 1) Discover (문제/대상/가설)
- 사용자 문제: 기존 사주 앱(점신, 포스텔러 등)은 두루뭉술한 템플릿 복붙으로 인해 재미 이상의 가치를 주지 못하며 결정적 순간에 도움을 주지 못함.
- 타겟 세그먼트:
  - B2B: 유저 결제 전환율을 높이려는 운세/커뮤니티/매칭 플랫폼.
  - B2C: 중대한 결정(퇴사, 창업, 이직, 결혼, 동업)을 앞두고 불안을 겪는 고관여자.
- 핵심 가설 3개:
  1. 유저는 단순한 '올해의 운세'보다 '5월 15일에 퇴사하라'는 구체적 행동 지시에 더 높은 비용을 지불할 것이다.
  2. B2B 고객사는 일반 운세 API보다 '특정 고민(이직, 재물) 해결용 API'를 더 비싼 단가에 도입할 것이다.
  3. LLM을 연동하여도, 코어 명리 엔진을 Vercel Edge에서 0.04ms로 처리하면 비용 Zero 구조를 유지할 수 있다.
- 실패 가설(버릴 조건): 만약 초기 B2B 고객사 또는 B2C 유저의 첫 결제 전환율이 기존 운세 앱 평균 이하이거나 유지보수 비용(LLM API 비용 포함)이 수익을 초과하면 피봇한다.

## 2) Strategy (포지셔닝/수익모델)
- 포지셔닝 한 줄: 인생의 중대한 결정 타이밍을 짚어주는 '행동 지시형' 라이프 코치 API.
- 경쟁 회피 전략: 귀여운 캐릭터나 방대한 운세 카테고리(점신 등)를 버리고, 오직 '결정(타이밍/궁합/재물)'에만 집중하여 고관여자 타겟팅.
- 수익 모델:
  - B2B: API 호출량 기반 종량제 과금 (Tier별 차등).
  - B2C: 프리미엄 1회성 결제 (예: 동업자 정밀 궁합, 이직 타이밍 리포트) 및 구독.
- 운영 자동화 원칙: Vercel Edge 배포, 무인 결제 및 키 발급 자동화, Watchdog을 통한 에러 자동 감지 및 텔레그램 알림. CS 제로화.

## 3) Write-PRD (MVP 범위 및 운영)
- MVP 범위(포함):
  - B2B용 코어 사주 연산 API 배포 (초경량 엔진).
  - LLM 프롬프트가 내장된 B2C/B2B 겸용 '행동 타이밍' 특화 엔드포인트 1~2개 (예: 이직운, 궁합운).
  - 인증/결제를 위한 Supabase + Stripe 최소 연동 (자동화).
- MVP 범위(제외): 화려한 UI/캐릭터 디자인, 방대한 잡다 운세(오늘의 운세 등 템플릿성).
- 성공 지표(KPI):
  - 첫 1주일 내 API 호출/결제 발생.
  - LLM 연동 포함 API 응답 속도 및 서버 비용 0원(또는 극소액) 유지.
- 출시 체크리스트:
  - 코어 엔진 Vercel Edge 배포 및 부하 테스트.
  - B2B API 문서(Swagger/GitBook) 작성.
  - B2C 랜딩페이지 (단건 결제 붙인 노코드/로우코드 페이지) 오픈.
- 리스크와 대응: LLM 할루시네이션으로 인한 엉뚱한 조언 -> 코어 명리 엔진의 결과값을 절대적 Guardrail로 삼는 프롬프트 엔지니어링 구축.
