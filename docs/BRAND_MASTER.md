> **문서 목적**: Vulcan Mission Control의 브랜드 아이덴티티, 네이밍 체계, 보이스, 비주얼 시스템을 정의하는 단일 진실 공급원(Single Source of Truth).

> **최종 갱신**: 2026-03-14 (v1.2)

> **문서 맵**: [PRODUCT_MASTER](PRODUCT_MASTER.md) · [BRAND_MASTER](BRAND_MASTER.md) · [ROADMAP](ROADMAP.md) · [WORK_PLAN](WORK_PLAN.md) · [PROGRESS](PROGRESS.md) · [DECISIONS](DECISIONS.md)

## (0) Assumptions

1. **단일 사용자**: 한 사람만 사용한다. 다국어, 접근성 등 다중 사용자 고려 불필요.
2. **Dark Only**: 라이트 모드 없음. 미션 컨트롤은 어두운 환경에서 사용한다.
3. **Desktop First**: 1920px 기준 설계, 모바일은 보조. 핵심 경험은 넓은 화면.
4. **에이전트 10개 (Pantheon 체계: Hermes + 9 Tier 2)**: Hermes, Aegis, Metis, Athena, Themis, Iris, Daedalus, Nike, Calliope, Argus. 확장 시 동일 네이밍 규칙 적용.
5. **OpenClaw 종속**: Vulcan의 브랜드 톤은 OpenClaw 전체 에코시스템과 일관되어야 한다.

## (1) Brand Foundation

### 1.1 One-liner

**에이전트가 일하는 동안, 조용히 불을 지키는 대장간.**

### 1.2 Essence

Vulcan은 대장장이의 화로(forge)다. 전령(Hermes)이 세상을 누비는 동안, 화로는 꺼지지 않고 모든 것을 기록하고, 벼리고, 비춘다. 소란스럽지 않게. 필요할 때 한 번 쳐다보면 모든 상황이 파악되도록.

### 1.3 Mission & Values

**Mission**
- AI 에이전트의 자율 활동에 투명한 가시성을 부여하여, 인간이 신뢰 위에서 결정할 수 있게 한다.

**Values**

| 가치 | 의미 |
|------|------|
| **Sovereignty** | 에이전트의 자율성을 침해하지 않는다 |
| **Transparency** | 숨기는 것 없이, 있는 그대로 보여준다 |
| **Calm** | 과장하지 않고, 필요한 것만 전한다 |
| **Warmth** | 기계적이지 않고, 사람 냄새가 난다 |
| **Trust** | 데이터를 지키고, 약속을 지킨다 |

### 1.4 Brand Promise

- 에이전트가 무엇을 하고 있는지, 앱을 열면 바로 보인다
- 어떤 에이전트가 어디에서 무엇을 하는지 오피스 메타포로 직관적으로 파악된다
- 쌓인 이벤트, 태스크, 메모리를 검색하고 추적할 수 있다
- 시스템이 꺼져도 에이전트는 멈추지 않는다 — Vulcan은 창이지 엔진이 아니다
- 불필요한 알림이나 개입 없이, 필요한 순간에만 정보를 제공한다

### 1.5 Positioning Statement

Vulcan은 생산성 도구도, 프로젝트 관리 앱도 아니다. Vulcan은 자율적으로 일하는 AI 에이전트의 활동을 한눈에 조망하는 관측소이자 지휘소다. 관찰과 제어를 통합하되, 에이전트의 자율성을 존중한다. 에이전트가 달리는 동안 인간은 이 화면을 통해 전체 그림을 보고, 필요할 때 결정을 내린다. 대장장이의 화로처럼 — 조용하고, 따뜻하고, 꺼지지 않는다.

### 1.6 Brand Personality

**Keywords (8)**

| 키워드 | 의미 |
|--------|------|
| **Calm** | 소란스럽지 않다. 정보를 제공하되 경고음을 울리지 않는다 |
| **Warm** | 어둡지만 차갑지 않다. 화로의 온기가 스며든다 |
| **Observant** | 모든 것을 보고 기록하되, 간섭하지 않는다 |
| **Trustworthy** | 데이터를 지키고, 보여주는 것이 사실이라는 확신을 준다 |
| **Minimal** | 필요한 것만 보여준다. 장식이나 과잉 정보가 없다 |
| **Grounded** | 과장 없이 현실적이다. 있는 그대로를 보여준다 |
| **Sovereign** | 에이전트의 독립성을 존중한다. 통제 욕구를 투영하지 않는다 |
| **Enduring** | 꺼지지 않는다. 화로처럼 묵묵히 타오른다 |

**Anti-keywords (5)**

| 피해야 할 것 | 이유 |
|--------------|------|
| Flashy / 화려한 | 관측소는 눈에 띄면 안 된다 |
| Commanding / 지시적 | 통제 도구가 아니다 |
| Cute / 귀여운 | 에이전트는 캐릭터가 아니다 |
| Anxious / 불안한 | 경고를 남발하면 신뢰가 깨진다 |
| Corporate / 기업적 | 한 사람을 위한 시스템이다 |

### 1.7 What Vulcan Will Never Do

1. 에이전트의 자율성을 무시하는 일방적 통제 UI를 만들지 않는다 — 제어는 확인 단계를 거쳐 신중하게
2. 경고 알림을 남발하거나, 불안감을 조성하는 메시지를 표시하지 않는다
3. 에이전트를 "부하직원"이나 "도구"로 표현하지 않는다 — 자율적 동료로 표현한다
4. 네온/사이버펑크/하이테크 미학을 차용하지 않는다
5. OpenClaw의 자율성을 위협하는 기능을 만들지 않는다

## (2) Naming Architecture — 신화 체계

### 2.1 체계 원칙

Vulcan 에코시스템의 모든 이름은 그리스-로마 신화에서 가져온다. 각 이름은 해당 요소의 역할과 성격을 상징적으로 반영한다.

### 2.2 네이밍 맵

| 이름 | 유래 | 상징 | 역할 |
|------|------|------|------|
| **Vulcan** | 로마 신화 — 불과 대장간의 신 (Hephaestus의 로마 대응) | 만드는 자, 벼리는 자. 보이지 않는 곳에서 기반을 다지는 장인 | Mission Control 시스템 이름 |
| **Hermes** | 그리스 신화 — 전령신, 경계의 신 | 경계를 넘나드는 메신저. 자율적이고 빠르다 | Chief of Staff / Orchestrator. 주 실행 에이전트 |
| **Aegis** | 그리스 신화 — 제우스/아테나의 방패 | 보호, 검증, 결함 방어 | Platform / Security / Reliability Lead |
| **Metis** | 그리스 신화 — 지혜의 여신, 제우스의 첫 아내 | 깊은 통찰, 정보의 정수를 추리는 자 | Research / Market Intel Lead |
| **Athena** | 그리스 신화 — 전략과 지혜의 여신 | 냉정한 판단, 전략적 배분 | Strategy / Finance / Portfolio Lead |
| **Themis** | 그리스 신화 — 정의와 질서의 여신 | 기준을 세우고 범위를 자르는 자 | Product Lead |
| **Iris** | 그리스 신화 — 무지개의 여신, 신들의 메신저 | 형태와 색채, 경험을 매개하는 자 | Design Lead |
| **Daedalus** | 그리스 신화 — 전설적인 장인, 미로의 건축가 | 실제로 만드는 자, 기술의 구현자 | Engineering / Automation Lead |

| **Argus** | 그리스 신화 — 백 개의 눈을 가진 거인 | 모든 것을 보고 평가하는 자, 증거를 요구하는 자 | Evaluator / QA / Eval Lead |
| **Hearth** | 화로 (영어) — Vulcan/Vesta의 불 | 대장간의 핵심. 따뜻함의 원천 | 시그니처 액센트 컬러 (`#e07a40`) |

### 2.3 레거시 네이밍 (Pantheon 확장 이전)

초기 에이전트 체계에서 사용되었으나 현재 Pantheon 10인 체계로 대체된 이름들:

### 2.4 확장 규칙

- PANTHEON.md에 소개문/슬로건/역할 카드를 추가한다

## (3) Visual Identity — Warm Obsidian

### 3.1 디자인 톤

**Warm Obsidian**: 흑요석(obsidian)처럼 어둡지만, 화로의 온기(warmth)가 스며드는 톤.

- 우주 관제실의 차가운 형광등이 아닌, 대장간 작업실의 따뜻한 어둠
- 정보 밀도는 높지만 시각적 피로는 낮아야 한다
- 색상은 적고, 의미 있는 곳에만 쓴다

## (4) Office Metaphor — 공간으로 읽는 상태

### 4.1 개요

에이전트의 상태를 사무실 공간에 매핑하여 직관적으로 파악한다. "Hermes가 Workbench에 있다" = "도구를 실행 중이다".

### 4.2 상태-공간 매핑

| 상태 | 오피스 존 | 시각적 의미 | 감정적 의미 |
|------|----------|------------|------------|
| `idle` | Watercooler | 휴식 공간에서 대기 | 여유, 다음 일감 기다림 |
| `writing` | Desk | 책상에서 작업 | 집중, 몰입 |
| `researching` | Library | 서재에서 조사 | 탐색, 분석 |
| `executing` | Workbench | 작업대에서 도구 사용 | 행동, 실행 |
| `syncing` | Hallway | 복도에서 이동/동기화 | 연결, 전달 |
| `error` | Red Corner | 경고 구역 | 주의, 확인 필요 |

### 4.3 시각적 표현

- **idle**: `--color-muted-foreground` (회색, 차분)
- **writing**: `--color-info` (파랑, 집중)
- **researching**: 보라 계열 (탐구적)
- **executing**: `--color-primary` (Hearth 오렌지, 불꽃 = 실행)
- **syncing**: `--color-success` (초록, 연결)
- **error**: `--color-destructive` (빨강, 주의)

### 4.4 활성 표시

- 비-idle 에이전트: `status-glow` CSS 효과 (해당 상태 색상의 box-shadow)
- Live Activity에서 15초 이내 이벤트: `activity-group-active` (Hearth 색 border + glow)
- `pulseWarm` 키프레임: Hearth 색상이 2.6초 주기로 숨쉬는 애니메이션

## (5) Voice & Tone

### 5.1 Global Voice Rules

Vulcan의 텍스트는 미션 컨트롤의 계기판 같아야 한다: 간결하고, 정확하고, 감정적이지 않되 차갑지 않다.

#### 문장 규칙

- **짧게**: 15자 내외. 20자를 넘지 않는다 (UI 텍스트 기준).
- **결론 먼저**: 상태 → 세부사항 순서.
- **능동태**: "저장됐어요" / "저장이 완료되었습니다" (X)
- **현재 시제**: "Hermes가 Workbench에 있어요" / "Hermes가 Workbench에 갔습니다" (X)

#### 따뜻함의 경계

**적절한 따뜻함**
- 에이전트 이름을 부른다 ("Hermes가 실행 중이에요")
- 여유를 준다 ("급하지 않으면 나중에 확인해도 돼요")
- 불확실하면 솔직하다 ("어댑터 연결을 확인해주세요")

**과잉 따뜻함 (금지)**
- 감탄사 ("우와!", "대단해요!")
- 과장 ("완벽하게 동기화됐어요!")
- 이모지
- 느낌표 2개 이상
- 에이전트를 캐릭터화 ("Hermes가 열심히 달리고 있어요~")

#### 확신 표현

| 수준 | 표현 |
|------|------|
| 확실 | ~이에요 / ~해요 |
| 추정 | ~로 보여요 / ~인 것 같아요 |
| 불확실 | 확인이 필요해요 |
| 모름 | 정보가 부족해요 |

### 5.2 에이전트 언급 규칙

에이전트는 자율적 동료다. 도구나 부하로 표현하지 않는다.

| 적절 | 부적절 |
|------|--------|
| "Hermes가 Workbench에 있어요" | "Hermes를 실행시켰어요" |
| "Atlas가 동기화 중이에요" | "Atlas한테 동기화 시켰어요" |
| "Aegis가 검증을 마쳤어요" | "Aegis가 검증 완료했습니다!" |
| "Lyra가 기억을 탐색 중이에요" | "Lyra에게 검색을 명령했어요" |

### 5.3 금지 어휘

| 어휘 | 이유 |
|------|------|
| 명령, 지시, 시킴 | 통제 언어 |
| 완벽, 최고, 대박 | 과장 |
| 걱정 마세요 | 과잉 위로 |
| 화이팅 | 펩토크 |
| !! (느낌표 2개) | 과잉 강조 |
| 이모지 전체 | 톤 불일치 |
| 실행시키다, 돌리다 | 에이전트를 도구로 취급 |
| 부하, 직원, 일꾼 | 에이전트를 종속적으로 표현 |

### 5.4 잘못된 톤 vs 올바른 톤 (5쌍)

**예시 1: 에이전트 상태**
- (X) "Hermes가 열심히 일하고 있어요! 화이팅!"
- (O) "Hermes — Workbench (executing)"

**예시 2: 에러 발생**
- (X) "앗! Hermes한테 문제가 생겼어요ㅠㅠ 걱정 마세요~"
- (O) "Hermes — Red Corner. 이벤트 로그를 확인해주세요."

## (7) Component Patterns

### 7.1 카드 (`vulcan-card`)

```css
border: 1px solid var(--color-border);
border-radius: var(--radius-card);        /* 12px */
background: var(--color-surface);
box-shadow: var(--shadow-soft);
```

용도: 에이전트 상태 카드, 메모리 카드, 문서 카드, 프로젝트 카드.

### 7.2 칩 (`vulcan-chip`)

```css
display: inline-flex;
border: 1px solid var(--color-border);
border-radius: 9999px;                    /* pill shape */
background: var(--color-muted);
color: var(--color-muted-foreground);
font-size: 0.75rem;
padding: 2px 10px;
```

용도: 태그, 상태 배지, 필터 칩.

### 7.3 버튼 (`vulcan-button`)

```css
border: 1px solid var(--color-primary);
border-radius: var(--radius-control);     /* 8px */
background: transparent;
color: var(--color-primary);
transition: background 180ms, color 180ms;
```

호버: `background: var(--color-primary); color: var(--color-background);`

### 7.4 입력 (`vulcan-input`)

```css
background: var(--color-muted);
border: 1px solid var(--color-border);
border-radius: var(--radius-control);     /* 8px */
color: var(--color-foreground);
```

포커스: `border-color: var(--color-primary);`

### 7.5 Sidebar 네비게이션

| 상태 | 텍스트 | 배경 | 특수 효과 |
|------|--------|------|----------|
| default | `--muted-foreground` | transparent | — |
| hover | `--foreground` | `--muted` | — |
| active | `--primary` | `--primary-12` | 좌측 3px accent bar (rounded-r-full) |
| active+hover | `--primary` | primary 16% | — |

## (8) Animation Language

### 8.1 원칙

- 애니메이션은 상태 변화를 전달하기 위해서만 사용한다
- 장식적 모션은 없다
- 빠르고 부드럽게: 180ms~360ms

### 8.2 키프레임

| 이름 | 용도 | 사양 |
|------|------|------|
| `fade-in-up` | 새 요소 진입 | translateY(12px→0), opacity(0→1), 360ms ease-out |
| `pulseWarm` | Hearth 색 숨쉬기 | box-shadow Hearth glow 0%→50%→100%, 2.6s infinite |

### 8.3 Framer Motion

- `AnimatePresence` + `motion.section/article`: 이벤트 리스트 진입/퇴장
- `initial={{ opacity: 0, y: 8 }}`, `animate={{ opacity: 1, y: 0 }}`
- `exit={{ opacity: 0 }}`, `transition={{ duration: 0.22 }}`

## (9) Anti-Patterns — 하지 않는 것

### 9.1 비주얼

| 금지 | 이유 |
|------|------|
| 네온 글로우, 사이버펑크 팔레트 | Warm Obsidian 톤과 충돌 |
| 밝은 배경 (라이트 모드) | Dark Only 원칙 |
| 과도한 그라디언트/블러 | 정보 가독성 저하 |
| 에이전트 캐릭터 일러스트 | 에이전트는 캐릭터가 아니라 존재 |
| 대시보드에 숫자/차트 과잉 | 메트릭스는 요약 카드 + 핵심 차트로 절제 |

### 9.2 인터랙션

| 안티패턴 | 올바른 방향 |
|----------|------------|
| 에이전트 제어를 캐주얼하게 노출 | 제어 액션(pause/resume/command)은 확인 단계(confirm)를 거쳐 신중하게 실행 |
| 승인 워크플로우를 무거운 UI로 구현 | Telegram 인라인 키보드 등 자연스러운 UX로 제공, Vulcan UI는 보조 패널 |
| 알림 폭탄 (Toast/Banner 남발) | Calm 원칙 — 중요 이벤트만 선별 알림 |
| 자동 새로고침으로 화면 점프 | WebSocket 실시간 업데이트는 현재 뷰를 방해하지 않는 방식으로 |

### 9.3 언어

| 금지 | 이유 |
|------|------|
| "AI가 대신 해드릴게요" | 과잉 약속 + 통제 언어 |
| "에이전트를 관리하세요" | Vulcan은 관측소이자 지휘소 — "관리"가 아닌 "관찰과 제어" |
| "실행 완료!" 감탄 표현 | 과잉 감정 |
| "에이전트 성능 보고서" | 기업적/평가적 언어 |
