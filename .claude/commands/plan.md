# /plan - Implementation Planning

복잡한 기능의 구현 계획을 수립합니다.

## 사용법
```
/plan [기능 설명]
```

## 프로세스

### 1. 요구사항 분석
- 기능 범위 파악
- 성공 기준 정의
- 제약사항 확인 (CLAUDE.md 금지사항 참조)

### 2. 아키텍처 검토
- 영향받는 파일 목록
- Drizzle 스키마 변경 여부
- SSE 스트림 영향 분석
- 기존 패턴 확인

### 3. 단계별 분해
- 구체적 작업 항목 + 파일 경로
- 의존성 순서
- 예상 복잡도 (S/M/L)

### 4. 실행 계획
- 순서 결정
- 검증 포인트 (lint → tsc → build)
- 위험 요소

## 예시
```
/plan SSE 연결 상태 표시 컴포넌트
```

## 자동 실행
- `planner` 에이전트 호출
- 필요시 `architect` 에이전트 추가 호출
